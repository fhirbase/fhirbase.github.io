Why FHIRbase?

Why choose FHIRbase for your next health IT project?



FHIR-native system

If you are building a FHIR system from scratch, FHIRbase provides you with a solid persistence layer:

  • Battle-proven & flexible design
  • Transactional consistency (ACID)
  • Storage of FHIR resources in JSONB
  • The power and expressiveness of the SQL language
  • Advanced indexing for better search performance

Analytics on FHIR

If you have a bunch of FHIR data and want to understand it, you can use FHIRbase as analytical tool

  • Load your data
  • Run analytical queries with SQL
  • Use the PostgreSQL ecosystem
