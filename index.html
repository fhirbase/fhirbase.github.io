Vision of FHIRbase

FHIRbase is excellent start and basis for creating standardized next generation Health IT solutions like EHRs, Patient Portals etc.
Using the FHIRbase, you can continue to use familiar technologies you (technology agnostic storage).

Heart of FHIRbase is PostgreSQL

PostgreSQL is a battle proved open source database which supports structured documents (jsonb) while preserving ACID guaranties and richness of SQL query language.

Soul of FHIRbase is FHIR standard

We decided to implement most of FHIR specification inside a database for scalability reason. This approach also gives you a possibility to use FHIRbase from your preferred lang/platform (.NET, java, ruby, nodejs etc).

Face of FHIRbase is Simple API

Most of API for FHIRbase is represented as functions in fhir schema, other schemas are used as code library modules.
